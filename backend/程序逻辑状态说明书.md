## 2. 逻辑状态定义

### 2.1 设备连接状态（核心状态机）

| 状态 | 条件 | 说明 |
|------|------|------|
| **DISCONNECTED** | `logged_in=False`, `socket=None`, IP不在 `api_pool` | 设备未连接 |
| **CONNECTING** | `socket` 正在建立，`logged_in=False` | 连接建立中 |
| **AUTHENTICATING** | `socket` 已建立，正在执行登录流程 | 认证中 |
| **CONNECTED** | `logged_in=True`, `socket` 有效, IP在 `api_pool` | 已连接已认证 |
| **OFFLINE** | `device_offline_flags[IP]=True`, 资源已清理 | 设备离线（不可恢复） |
| **RECONNECTING** | `logged_in=False`, 正在尝试重新登录 | 重连中 |

### 2.2 WebSocket 连接状态

| 状态 | 条件 | 说明 |
|------|------|------|
| **WS_INIT** | WebSocket已建立，未收到首条消息 | 等待初始化 |
| **WS_ACTIVE** | 已注册到 `active_connections`, keepalive任务运行中 | 活跃连接 |
| **WS_POLLING** | 至少一个轮询任务在 `polling_tasks` 中运行 | 正在轮询数据 |
| **WS_CLOSING** | 收到stop命令或WebSocket关闭 | 正在关闭 |
| **WS_CLOSED** | `is_ws_closed()` 返回True, 资源已清理 | 已关闭 |

### 2.3 轮询任务状态

| 状态 | 条件 | 说明 |
|------|------|------|
| **POLL_IDLE** | 无轮询任务在运行 | 空闲 |
| **POLL_RUNNING** | 对应任务在 `polling_tasks` 中且未done | 正在轮询 |
| **POLL_ERROR** | `consecutive_errors >= MAX_CONSECUTIVE_ERRORS` | 错误超限 |
| **POLL_STOPPED** | stop_event被set或任务被cancel | 已停止 |

### 2.4 流量监控状态

| 状态 | 条件 | 说明 |
|------|------|------|
| **TRAFFIC_IDLE** | `traffic_managers` 中无该IP | 无流量监控 |
| **TRAFFIC_MONITORING** | `running=True`, `paused=False` | 正常监控 |
| **TRAFFIC_PAUSED** | `running=True`, `paused=True` | 暂停发送 |
| **TRAFFIC_RECONNECTING** | 连接失效，正在重连 | 重连中 |
| **TRAFFIC_STOPPED** | `running=False` | 已停止 |

### 2.5 设备重启状态

| 状态 | 条件 | 说明 |
|------|------|------|
| **REBOOT_IDLE** | `rebootModalVisible=false`, `rebootLoading=false` | 空闲，未触发重启 |
| **REBOOT_CONFIRMING** | `rebootModalVisible=true`, `rebootLoading=false` | 显示重启确认弹窗 |
| **REBOOT_EXECUTING** | `rebootLoading=true` | 正在发送重启命令 |
| **REBOOT_SENT** | 后端返回 success，前端显示成功提示 | 重启命令已发送，设备即将离线 |

### 2.6 终端会话状态

| 状态 | 条件 | 说明 |
|------|------|------|
| **TERM_DISCONNECTED** | `isConnected=false`, `isConnecting=false`, wsRef=null | 终端未连接 |
| **TERM_CONNECTING** | `isConnecting=true`, WebSocket正在打开 | 正在建立WebSocket连接 |
| **TERM_AUTHENTICATING** | WebSocket已建立，正在等待后端Telnet登录完成 | Telnet登录认证中 |
| **TERM_CONNECTED** | `isConnected=true`, Telnet会话已建立 | 终端已连接，可输入命令 |
| **TERM_RECONNECTING** | WebSocket意外断开，前端显示断开状态 | 需要用户手动重新连接 |

## 3. 状态转换

### 3.1 设备连接状态转换

| 从状态 | 到状态 | 触发代码 | 条件 | 变量修改 |
|--------|--------|---------|------|---------|
| DISCONNECTED | CONNECTING | `MikroTikAPI.__init__()` + `login()` | 用户发起连接 | `socket` 创建 |
| CONNECTING | AUTHENTICATING | `_try_legacy_login()` | socket.connect()成功 | `socket` 有效 |
| AUTHENTICATING | CONNECTED | `_plaintext_login()` / `_challenge_login()` | 登录响应 `!done` | `logged_in=True`, `api_version` 设置 |
| AUTHENTICATING | DISCONNECTED | 登录失败 | 响应含 `failure`/`bad`/`trap` | `socket` 关闭, `logged_in=False` |
| CONNECTED | OFFLINE | `mark_device_offline()` | 心跳连续失败/连接异常 | `device_offline_flags[IP]=True`, 清理所有资源 |
| CONNECTED | RECONNECTING | 各监控循环中检测到连接断开 | `logged_in=False` 或 socket异常 | 尝试重新login |
| RECONNECTING | CONNECTED | 重连login成功 | `login()` 返回True | `logged_in=True`, `reconnect_count=0` |
| RECONNECTING | OFFLINE | 重连次数达上限 | `reconnect_count > MAX_RECONNECT_ATTEMPTS` | 标记离线 |

### 3.2 WebSocket 状态转换

| 从状态 | 到状态 | 触发代码 | 条件 | 变量修改 |
|--------|--------|---------|------|---------|
| WS_INIT | WS_ACTIVE | `register_connection()` | 首条消息含有效IP | `active_connections[IP]` 添加WS |
| WS_ACTIVE | WS_POLLING | `handle_interface_polling_single_conn()` 等 | 收到start_polling请求 | `polling_tasks` 添加任务 |
| WS_POLLING | WS_ACTIVE | stop事件被set | 收到stop命令 | 取消任务, 从 `polling_tasks` 移除 |
| WS_ACTIVE | WS_CLOSING | WebSocket断开或收到stop | 连接关闭/stop命令 | stop_events全部set |
| WS_CLOSING | WS_CLOSED | `unregister_connection()` + finally块 | 所有清理完成 | 从 `active_connections` 移除 |

### 3.3 流量监控状态转换

| 从状态 | 到状态 | 触发代码 | 条件 | 变量修改 |
|--------|--------|---------|------|---------|
| TRAFFIC_IDLE | TRAFFIC_MONITORING | `TrafficMonitorManager.start_monitor()` | 接口轮询启动 | `running=True`, 创建监控线程 |
| TRAFFIC_MONITORING | TRAFFIC_PAUSED | `TrafficMonitorManager.pause()` | 收到pause_traffic动作 | `paused=True` |
| TRAFFIC_PAUSED | TRAFFIC_MONITORING | `TrafficMonitorManager.resume()` | 收到resume_traffic动作 | `paused=False` |
| TRAFFIC_MONITORING | TRAFFIC_RECONNECTING | `_start_monitor_sync()` 内层循环 | 连续超时>=3 | 关闭旧socket, 尝试login |
| TRAFFIC_RECONNECTING | TRAFFIC_MONITORING | 重连成功 | login成功 | `consecutive_timeouts=0` |
| TRAFFIC_RECONNECTING | TRAFFIC_STOPPED | 重连失败达上限 | `attempt >= max_reconnect_attempts` | `running=False`, 调用 `mark_device_offline()` |
| TRAFFIC_MONITORING | TRAFFIC_STOPPED | `stop_monitor()` / `cleanup_device_resources()` | 用户登出/设备离线 | `running=False`, 关闭API |

## 4. 控制流骨架

### 4.1 程序启动流程

```
launcher.py:main()                          # 启动器（用户双击的入口）
  ├── kill_old_backend_processes()          # 终止旧版本后端进程
  ├── is_port_open(32995)                   # 检测后端是否已运行
  │   ├── 已运行 → is_process_running('shunlian_backend.exe')
  │   │   ├── 是 → 进入守护模式
  │   │   └── 否 → kill_backend() + start_backend() + wait_for_backend()
  │   └── 未运行 → start_backend() + wait_for_backend()
  ├── open_browser()                        # 打开浏览器访问前端
  └── guardian_loop()                       # 进入进程守护循环
       └── [循环] 每10秒:
            ├── is_process_running()         # 检测后端进程是否存活
            ├── is_port_open(32995)          # 检测端口是否监听
            ├── check_health_api()           # 请求 /api/health
            ├── 全部正常 → consecutive_failures=0
            └── 连续3次失败 → restart_backend()
                 ├── kill_backend()          # 终止旧进程
                 ├── start_backend()         # 重新启动
                 ├── wait_for_backend()      # 等待就绪
                 └── 频率限制: 5分钟内最多5次重启

main.py:main()                              # 后端主入口（由launcher启动）
  ├── load_config()                         # 加载 config.yaml
  ├── setup_logging()                       # 配置日志
  ├── check_admin()                         # 检查管理员权限
  └── uvicorn.run(app, ...)                 # 启动 FastAPI
       └── api_server.py:lifespan()         # 应用生命周期
            ├── _terminate_unauthorized_helper()   # 终止ct_helper.exe
            ├── _start_ct_helper_monitor()         # 启动监控线程
            ├── mndp_core.start_listener()         # 启动MNDP UDP监听
            ├── mndp_core.start_auto_discover()    # 启动MNDP自动发现(10s间隔)
            └── run_websocket_server()             # 在新线程启动WebSocket
                 └── websockets.serve(handle_websocket, port=32996)
```

### 4.2 WebSocket 主消息处理

```
handle_websocket(websocket)
  ├── 等待首条消息 → 解析 {ip, username, password, flags...}
  ├── register_connection(websocket, device_ip)
  │
  ├── 分支判断（互斥）:
  │   ├── is_interface_polling → handle_interface_polling_single_conn()
  │   │    ├── 创建 MikroTikAPI + login
  │   │    ├── 创建 TrafficMonitorManager + start_send_task()
  │   │    └── 启动 interface_polling_task_with_traffic_single_conn()
  │   │         └── 循环: get_interface_list() → 推送 → sleep(2s)
  │   │              └── 内嵌: websocket.recv(timeout=2s) 处理IP操作
  │   │
  │   ├── is_wireless_interfaces → handle_wireless_polling_single_conn()
  │   ├── is_wireless_clients → handle_wireless_clients_single_conn()
  │   ├── is_security_profiles → handle_security_profiles_single_conn()
```

### 4.3 Keepalive 心跳

```
keepalive_task(device_ip, username, password, websocket)
  ├── 创建独立 MikroTikAPI + login
  │    └── 失败 → mark_device_offline() → return
  │
  └── 循环 (每3秒):
       ├── 检查 is_ws_closed(websocket) → 是则 break
       ├── heartbeat_api.keepalive_check()
       │    ├── 成功 → consecutive_failures = 0
       │    └── 失败 → consecutive_failures += 1
       │         └── >= 5 → send_device_offline() + mark_device_offline() → break
       └── 发送 {'action': 'ping'} 到前端
```

### 4.6 设备离线检测（多路径）

```
路径1: Keepalive心跳
  keepalive_task → consecutive_failures >= 5
    → send_device_offline() → mark_device_offline()

路径2: 接口轮询
  interface_polling_task_with_traffic_single_conn
    → consecutive_errors >= MAX_CONSECUTIVE_ERRORS
    → 推送 device_offline → mt_api.logged_in = False

路径3: HTTP API异常
  _check_connection_error() → 检测到 10054/reset/refused/timed out
    → 清理 api_pool → mark_device_offline()

路径4: 流量监控
  TrafficMonitorManager._start_monitor_sync()
    → 重连失败达上限 → mark_device_offline()

路径5: 设备重启（预期内离线）
  /api/system/reboot → 发送 /system/reboot 命令
    → 设备开始重启，TCP连接断开
    → read_sentence 抛出 ConnectionResetError/OSError
    → 捕获异常，视为正常现象，返回 success
    → 前端显示成功提示 → keepalive检测到离线 → 触发重连弹窗

mark_device_offline(ip):
  ├── 设置 device_offline_flags[ip] = True (幂等)
  ├── cleanup_device_resources(ip)
  │    ├── 停止 traffic_managers
  │    ├── 取消 interface_polling_tasks
  │    ├── 关闭 interface_api_connections
  │    ├── 关闭 device_api_connections
  │    ├── 取消 device_watch_tasks
  │    ├── 关闭 log_api_connections
  │    ├── 清理 log_cache_store
  │    ├── 清理 connection_filters
  │    ├── 清理 device_last_activity
  │    ├── 清理 device_download_status
  │    └── 关闭所有 WebSocket 连接
  ├── 清理 api_pool[ip]
  └── 向所有 WS 发送 {'status': 'device_offline'}
```

```
  │
```

```
  ├── term.onData: sendInput(data)

  ┌─────────────────────────────────────────────────────────┐
  │ handle_terminal_session(websocket, device_ip, username, password)
  │
  │  ├── 1. 创建 session_id → terminal_sessions[session_id] = {}
  │  │
  │  ├── 2. Telnet 连接 (asyncio):
  │  │    ├── asyncio.open_connection(device_ip, 23)  # Telnet端口23
  │  │    ├── telnet_login():
  │  │    │    ├── 读取 Login: 提示 → 发送 username
  │  │    │    ├── 读取 Password: 提示 → 发送 password

  │  │    │
  │  │    ├── 3. 发送初始化命令:
  │  │    │    ├── send("/terminal/ink")
  │  │    │    ├── send("/terminal/screen")
  │  │    │    └── send("/terminal/word")
  │  │    │
  │  │    └── 4. 发送 terminal_init 消息 → 前端:
  │  │         └── onmessage: { type: 'terminal_init', message: '' }
  │  │
  │  ├── 5. 主循环 (并行两个任务):
  │  │    ├── Task A - 读取 Telnet 输出:
  │  │    │    └── 循环: reader.read(4096)
  │  │    │         ├── 解码: decode_mikrotik_hex_escape(data)
  │  │    │         ├── 过滤 Telnet 协议控制字符 (IAC, DONT, DO, WILL, WONT, SB, SE)
  │  │    │         └── 发送 { type: 'terminal_output', data } → 前端
  │  │    │
  │  │    └── Task B - 接收 WebSocket 输入:
  │  │         └── 循环: websocket.receive_text()
  │  │              ├── 解析 action
  │  │              ├── 'terminal_input' → Telnet writer.write(data)

  │  │
  │  └── 6. 清理:
  │       ├── 取消所有任务
  │       ├── Telnet writer.close()

  │       └── 发送 { type: 'terminal_output', data: '\r\n会话已终止\r\n' }

       ├── 提取十六进制字节序列
       ├── bytes.fromhex(hex_str).decode('utf-8', errors='replace')
       └── 返回解码后文本 (正确显示中文)

MikroTik 设备行为:
  ├── 支持 / 开头的 CLI 命令 (如 /interface print)
  ├── 支持普通命令 (如 ping 8.8.8.8)

  ├── 输出包含十六进制转义 (<XX XX XX>)
  └── 会话保持: 持续到用户登出或 WebSocket 断开
```

## 7. New Terminal 终端控制流

### 7.1 终端连接流程

```
架构概述:
  前端 (TerminalPage.tsx) → TerminalContext.tsx (状态管理) → WebSocket → 后端 (websocket_server.py) → Telnet → MikroTik设备
                                                                    ↑
                                                          bufferRef (输出持久化)
  ─────────────────────────────────────────────────────────────────────

前端 TerminalContext (全局单例，跨页面切换保持)
  ├── connectTerminal(router):
  │    ├── 关闭已有连接 (disconnectTerminal)
  │    ├── 创建新 WebSocket → ws://host/ws
  │    ├── isConnecting = true
  │    ├── onopen:
  │    │    └── 发送 { action: 'terminal_connect', ip, username, password }
  │    ├── onmessage:
  │    │    ├── { type: 'terminal', status: 'connected' } → isConnected=true, isConnecting=false
  │    │    ├── { type: 'terminal', status: 'output', data } → bufferRef.current += data
  │    │    │                                    → 调用 replayCallback(data)
  │    │    └── { type: 'terminal', status: 'error', message } → 显示错误, disconnectTerminal
  │    └── onclose:
  │         └── isConnected=false, 保留 bufferRef 内容
  │
  ├── disconnectTerminal():
  │    ├── 关闭 WebSocket (wsRef.current.close())
  │    ├── isConnected=false, wsRef=null
  │    ├── bufferRef='' (清空输出缓冲)
  │    └── 后端自动清理 Telnet 会话
  │
  ├── sendInput(data):
  │    └── wsRef.current.send({ action: 'terminal_input', data })
  │
  ├── onReplayBuffer(callback):
  │    └── 注册 replayCallback, 立即回放 bufferRef 内容
  │
  └── getBuffer():
       └── 返回 bufferRef.current (切换到后端获取全部缓冲)

前端 TerminalPage (UI组件)
  ├── 挂载时: onReplayBuffer → term.write(data)
  ├── term.onData: sendInput(data)
  ├── 清理: term.dispose()
  └── 不管理连接生命周期（由 TerminalContext 管理）

后端 WebSocket 终端处理流程:
  ┌─────────────────────────────────────────────────────────┐
  │ handle_terminal_session(websocket, device_ip, username, password)
  │
  │  ├── 1. 创建 session_id → terminal_sessions[session_id] = {}
  │  │
  │  ├── 2. Telnet 连接 (asyncio):
  │  │    ├── asyncio.open_connection(device_ip, 23)  # Telnet端口23
  │  │    ├── telnet_login():
  │  │    │    ├── 读取 Login: 提示 → 发送 username
  │  │    │    └── 读取 Password: 提示 → 发送 password
  │  │    └── 登录成功 → 发送 { type: 'terminal', status: 'connected' }
  │  │
  │  ├── 3. 数据转发循环:
  │  │    ├── 前端 → 后端: terminal_input action → 写入 Telnet
  │  │    ├── Telnet → 前端: 读取输出 → { type: 'terminal', status: 'output', data }
  │  │    └── 异常处理: Telnet断开 → 发送 error 状态
  │  │
  │  └── 4. 清理: finally块 → 关闭 Telnet → 删除 terminal_sessions[session_id]
  └─────────────────────────────────────────────────────────┘
```

### 7.2 终端自动连接/重连机制

```
前端 TerminalPage.tsx
  ├── 自动连接:
  │    ├── 进入终端页面时检测 router?.ipAddress && !deviceOffline
  │    ├── autoConnectedRef.current = false → 触发 connectTerminal()
  │    └── autoConnectedRef.current = true → 防止重复连接
  │
  ├── 设备离线处理:
  │    ├── deviceOffline = true → autoConnectedRef.current = false
  │    ├── TerminalContext 检测 deviceOffline → disconnectTerminal()
  │    └── 清空输出缓冲，关闭 WebSocket
  │
  └── 设备重连处理:
       ├── deviceOffline = false && !wsRef.current → 延迟1秒后自动重连
       ├── connectTerminal(routerRef.current) → 重新建立连接
       └── bufferRef 保留历史输出（用户可见完整会话记录）

前端 TerminalContext.tsx
  ├── 设备离线自动断开:
  │    ├── useEffect 监听 deviceOffline
  │    └── deviceOffline = true → disconnectTerminal()
  │
  └── 设备重连自动恢复:
       ├── useEffect 监听 deviceOffline 变化
       ├── !deviceOffline && routerRef.current?.ipAddress && !wsRef.current
       └── setTimeout(1000ms) → connectTerminal(routerRef.current)
```
