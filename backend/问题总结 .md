# 瞬联调式工具 问题总结

## 当期问题

### 1. Nstreme 标签栏功能实现（已解决）

**问题描述**：WinboxEditor 中的 "Nstreme" 标签栏缺少对应功能，需参考 Winbox 中的 Nstreme 标签栏实现。

**根本原因**：Nstreme 相关字段位于独立子菜单 `/interface/wireless/nstreme`，而非 `/interface/wireless` 主菜单，原后端未支持该子菜单的读取和更新。

**解决方案**：
- 后端 `api_server.py`：
  - `get_wireless_interface` 端点新增 Nstreme 子菜单查询，使用 `.proplist=.id,name,enable-nstreme,framer-policy,framer-limit,enable-polling,disable-csma`
  - `update_wireless_interface` 端点新增 Nstreme 字段更新逻辑，通过 `/interface/wireless/nstreme/set` 命令应用配置
  - `WirelessInterfaceUpdateRequest` 模型新增 5 个 Nstreme 字段：`enable_nstreme`、`nstreme_framer_policy`、`nstreme_framer_limit`、`nstreme_enable_polling`、`nstreme_disable_csma`
- 前端 `WinboxEditor.tsx`：实现 `renderNstremeTab()` 函数，包含 Nstreme 配置区（启用 Nstreme、启用轮询、禁用 CSMA）和帧聚合区（帧打包策略、帧大小上限）
- 前端 `WirelessPage.tsx`：`getChangedFields()` 添加 5 个 Nstreme 子菜单字段映射

**验证结果**：端到端 API 测试通过（读取、更新、验证、还原全部成功）。

### 2. get_wireless_interface 端点查询语法 Bug（已解决）

**问题描述**：`get_wireless_interface` 端点查询接口详情时返回"接口不存在"。

**根本原因**：后端使用 `=.id=*1` 语法查询特定接口，但 RouterOS 6.47.9 不支持该语法，返回 `unknown parameter` 错误。正确语法应为 `?.id=*1`（查询过滤语法）。

**解决方案**：
- `api_server.py:1717` 将 `=.id={interface_id}` 改为 `?.id={interface_id}`
- `api_server.py:2038` `get_wireless_frequency_info` 端点同步修复相同问题

**影响范围**：此为预先存在的 Bug，影响所有无线接口详情查询（编辑弹窗加载时），前端通过 fallback 到 plural 端点规避了该问题，但无法获取完整字段。

### 3. 无线标签栏默认显示及字段重命名（已解决）

**问题描述**：
1. 无线配置弹窗默认显示"常规"标签栏，应默认显示"无线"标签栏
2. "无线"标签栏内"频段"文本应改为"物理协议"
3. "无线电名称"文本应改为"射频接口名称"

**解决方案**：
- `WinboxEditor.tsx:510` `useState('general')` 改为 `useState('wireless')`
- `WinboxEditor.tsx:1085` 标签"频段"改为"物理协议"
- `WinboxEditor.tsx:1163` 标签"无线电名称"改为"射频接口名称"，placeholder 同步修改

### 4. 终端漫游（station-roaming）功能实现（已解决）

**问题描述**："无线-基本配置"标签栏中缺少"终端漫游"功能，对应 Winbox 中的 station-roaming 功能。

**根本原因**：后端 proplist 未包含 `station-roaming` 字段，`interface_data` 字典也未提取该字段。

**解决方案**：
- 后端 `api_server.py`：
  - `get_wireless_interfaces`（plural）和 `get_wireless_interface`（singular）端点的 `.proplist` 添加 `station-roaming`
  - `interface_data` 字典添加 `'station-roaming': iface.get('station-roaming', '')`
  - `WirelessInterfaceUpdateRequest` 模型新增 `station_roaming: str = ""` 字段
  - `update_wireless_interface` 端点新增 `=station-roaming={request.station_roaming}` 命令构建
- 前端 `WinboxEditor.tsx`：在"无线加密"行同一行添加"终端漫游"下拉框，仅在客户端模式（station / station-bridge / station-pseudobridge / station-wds）下显示；非客户端模式下"无线加密"下拉框限制最大宽度为 `calc(50% - 8px)`，不占整行
- 前端 `WirelessPage.tsx`：`getChangedFields()` 添加 `station_roaming` 字段映射

**验证结果**：端到端 API 测试通过（读取、启用、禁用、还原 station-roaming 全部成功）。
