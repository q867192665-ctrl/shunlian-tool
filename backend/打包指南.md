#### 步骤2: 更新 `更新日志.md`

1. 打开 `问题总结.md` 文件，查看"## 当期问题"部分的所有内容
2. 根据当期问题提炼生成新版本的更新日志，日志内容偏向普通用户，应包含以下分类：
   - **新增功能**：新功能、新工具集成等
   - **优化改进**：性能优化、流程改进、参数调整等
   - **问题修复**：修复的各类问题
   - **UI 优化**：界面优化、图标调整等
3. **重要规则**：
   - 更新日志内容必须严格从"问题总结"中提炼，问题总结中没有的内容不要写入
   - 更新日志中不要出现任何暴露设备品牌信息的文本（如 MikroTik、RouterOS 等），面向普通用户
4. 打开 `更新日志.md` 文件，**替换整个文件内容为当前版本的日志记录**（不保留往期历史版本）：

- 安装包内的 `更新日志.md` 只展示当前版本的更新内容，往期版本对用户无参考价值
- 历史版本日志可在 GitHub 仓库的 Release 页面查看
- 文件体积更小，减少安装包体积

往期版本日志通过 GitHub Release 页面查看。

#### 步骤4：整理文件
```powershell
$setupDir = "dist\setup_files"
New-Item -ItemType Directory -Force -Path $setupDir
Copy-Item "dist\shunlian_backend.exe" $setupDir
Copy-Item "dist\ShunLianTool.exe" $setupDir
Copy-Item "config.yaml" $setupDir
Copy-Item "logo.ico" $setupDir
Copy-Item "Logo.jpg" $setupDir
if (Test-Path "SLSCtools.exe") { Copy-Item "SLSCtools.exe" "$setupDir\slsc_runtime.slr" }
if (Test-Path "$setupDir\static") { Remove-Item -Recurse -Force "$setupDir\static" }
Copy-Item -Recurse -Force "static" "$setupDir\static"
Copy-Item -Force "更新日志.md" "$setupDir\"
if (Test-Path "iperf3") { Copy-Item -Recurse -Force "iperf3" "$setupDir\iperf3" }
if (Test-Path "autodefaultport.rsc") { Copy-Item -Force "autodefaultport.rsc" "$setupDir\slsc_data.sld" }
```

**⚠️ 重要提示（常见错误）**：
`更新日志.md` 必须在整理文件步骤的最后复制。**PowerShell 的 `Copy-Item` 在特定情况下可能不会真正覆盖文件**（即使加了 `-Force`），具体表现为：
- 源文件和目标文件内容不同，但复制后目标文件内容未更新
- 安装后程序版本号或更新日志显示旧版本内容

**预防措施**：
1. 复制后必须进行验证：`Get-Content "dist\setup_files\更新日志.md" -TotalCount 3`
2. 使用显式参数写法，确保正确覆盖：
   ```powershell
   Copy-Item -Force -Path "更新日志.md" -Destination "dist\setup_files\更新日志.md"
   ```
3. 如果打包后发现问题，需要：
   - 手动验证：`Get-Content -Encoding UTF8 "dist\setup_files\更新日志.md" -TotalCount 3`
   - 确认第一行版本号正确后，重新运行 Inno Setup 编译

## 六、关键代码注意事项

### 1. `get_base_dir()` — 所有文件必须使用
PyInstaller 打包后 `__file__` 指向临时目录，必须用此函数获取exe所在目录：
```python
def get_base_dir():
    if getattr(sys, 'frozen', False):
        return os.path.dirname(sys.executable)
    return os.path.dirname(os.path.abspath(__file__))
```
**涉及文件**：`main.py`、`launcher.py`、`api_server.py`、`websocket_server.py`

### 2. `NullWriter` — --noconsole 模式必须
`--noconsole` 时 `sys.stdout`/`sys.stderr` 为 None，uvicorn 调用 `.isatty()` 会崩溃：
```python
if getattr(sys, 'frozen', False):
    class NullWriter:
        def write(self, *args): pass
        def flush(self): pass
        def isatty(self): return False
    if sys.stdout is None:
        sys.stdout = NullWriter()
    if sys.stderr is None:
        sys.stderr = NullWriter()
```
**涉及文件**：`main.py` 的 `main()` 函数开头

### 3. 日志写入 LOCALAPPDATA — 避免权限错误
安装到 Program Files 后无法写日志，必须写到用户目录：
```python
if getattr(sys, 'frozen', False):
    local_app = os.environ.get('LOCALAPPDATA', '')
    if local_app:
        log_dir = os.path.join(local_app, 'ShunLianTool')
    else:
        log_dir = get_base_dir()
    os.makedirs(log_dir, exist_ok=True)
```
**日志路径**：`%LOCALAPPDATA%\ShunLianTool\瞬联调试工具.log`、`%LOCALAPPDATA%\ShunLianTool\launcher.log`

### 4. PyInstaller 参数
- `--noconsole`：不显示CMD黑窗口
- `--onefile`：打包为单个exe
- `--noconfirm`：覆盖已有输出
- `--hidden-import`：uvicorn/websockets/routeros_api 等动态导入需要手动声明
- `--collect-all asyncio`：Win7 必需，收集 asyncio C 扩展

## 九、常见问题

### 通用问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `PermissionError` 写日志 | 安装到 Program Files 无写权限 | 日志写 LOCALAPPDATA |
| `AttributeError: 'NoneType'.isatty()` | --noconsole 下 stderr 为 None | NullWriter 替换 |
| `{"detail":"Frontend not found"}` | __file__ 在 PyInstaller 临时目录 | 用 get_base_dir() |
| Inno Setup 静默失败 | build_all.bat 路径含中文 | 手动运行 ISCC.exe |
| **安装后页面纯白** | vite.config.ts 的 base 不是 `/static/` | 修改 base 为 `/static/` 后重新 build |
| **Inno Setup 编译失败：找不到 wizmodernimage-is.bmp** | 新版本 Inno Setup 已移除 bmp 向导图片 | 删除 setup.iss 中的 `WizardImageFile` 和 `WizardSmallImageFile` 配置 |
| **Inno Setup 编译失败：更新日志.md 不存在** | 打包脚本未复制该文件 | 确保 `build_all.bat` 中有 `copy 更新日志.md` 命令 |
| **安装后更新日志显示旧版本** | 打包过程中 `更新日志.md` 未被正确覆盖（PowerShell `Copy-Item` 忽略新版本） | 1. 手动复制：`Copy-Item -Force -Path "更新日志.md" -Destination "dist\setup_files\更新日志.md"` 2. 验证：`Get-Content -Encoding UTF8 "dist\setup_files\更新日志.md" -TotalCount 5` 3. 确认第一行版本号正确后重新编译 |
| **安装后登录页面显示版本号为 v1.0.0** | `/api/app-version` 从 `setup.iss` 读取版本号，但 PyInstaller 打包时 `setup.iss` 不在 exe 内 | 已修复为从 `更新日志.md` 中动态解析版本号（`_parse_current_version()`） |
| **安装后登录页面版本号与当前版本不符（显示旧版本）** | `dist\setup_files\更新日志.md` 中的版本号是旧的，未正确覆盖 | 修复方法同"安装后更新日志显示旧版本"，验证文件并重新编译 |

### Win7 SP1 专属问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `ValueError: Unrecognised argument(s): encoding` | `logging.basicConfig(encoding='utf-8')` 是 Python 3.9+ 语法 | 用 `codecs.open()` + `StreamHandler` 代替 |
| `NameError: name 'SSLContextFactory' is not defined` | `X \| None` 联合类型语法是 Python 3.10+ 语法 | 用 `Optional[X]` 代替（`from typing import Optional`） |
| `ModuleNotFoundError: No module named '_overlapped'` | PyInstaller 未收集 Windows asyncio C 扩展 | 后端打包加 `--collect-all asyncio` |
| 错误弹窗 | 子进程 stderr 未重定向 | launcher.py 中重定向 stdout/stderr 到日志文件 |
| `Failed to remove temporary directory` 警告 | PyInstaller 退出时清理临时目录失败 | 无害警告，可忽略 |

### Python 3.8 兼容性清单

以下语法在 Python 3.8 中**不可用**，必须替换：

| Python 3.9+ 语法 | Python 3.8 替代 |
|-------------------|-----------------|
| `logging.basicConfig(encoding='utf-8')` | `codecs.open()` + `StreamHandler` |
| `X \| None`（联合类型） | `Optional[X]`（`from typing import Optional`） |
| `dict \| dict`（字典合并） | `{**d1, **d2}` 或 `d1.copy(); d1.update(d2)` |
| `list[int]`（内置泛型） | `List[int]`（`from typing import List`） |

**本项目已处理 Python 3.8 兼容性**：
- `main.py`、`launcher.py`：已使用 `codecs.open()` 代替 `encoding` 参数
- `websocket_server.py`、`ssl_context.py`：已添加 `from __future__ import annotations`，支持 `X | None` 语法

## 十、版本管理

### 版本号规范

版本号格式：`v主版本.次版本.修订版本`（如 `v1.0.0`）

- **主版本**：重大功能更新或架构变更
- **次版本**：新增功能或较大改进
- **修订版本**：Bug修复、UI优化、小改进

### 版本相关文件

| 文件 | 说明 |
|------|------|
| `setup.iss` | `#define AppVersion "1.0.0"` - 安装包版本号 |
| `更新日志.md` | **仅保留当前版本更新内容**，往期版本历史通过 GitHub Release 页面查看 |

### 发布新版本流程

**步骤1：更新版本号**
   - 修改 `setup.iss` 中的 `AppVersion`

**步骤2：打包发布**
   - 按照本指南第三部分**打包前准备**，**替换** `更新日志.md` 为当前版本内容
   - 按照本指南第四部分进行打包
   - 测试安装包功能
   - 发布到分发渠道
   - 在 GitHub 创建 Release，填写完整更新日志
